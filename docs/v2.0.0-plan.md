# MathGame v2.0.0 — Plan de release

> **TL;DR.** Convertir MathGame de un menú de 6 modos sin estado en una app premium multiplataforma (iPhone · iPad · Apple Watch · Mac) con un motor de **progresión diaria** (reto, racha, XP, achievements, Game Center) y polish AAA (sonido, haptics, animaciones, accesibilidad). Modelo de negocio se mantiene: **pago único** en App Store, con un salto de precio justificado por el cambio mayor.
>
> **Duración**: 8 semanas calendario, 7 fases. Cada fase termina con un build instalable en TestFlight.

---

## 2. Pilares

1. **Progresión diaria** — Daily Challenge, racha persistente, XP, niveles, achievements internos + Game Center.
2. **Profundidad sin perder simplicidad** — 4 modos nuevos sin saturar el menú.
3. **Polish premium** — sonido, haptics, animaciones spring, tipografía y paleta refinadas, onboarding.
4. **Multi-plataforma de verdad** — layouts adaptativos reales, no sólo "compatible".
5. **Accesibilidad real** — VoiceOver, Dynamic Type, contraste, opción de reducir movimiento.

## 3. Arquitectura técnica

### 3.1 Lo que se elimina

- **Duplicación de `GameViewModel`**: existe en `Presentation/Pages/CalculationView.swift` (compilado) y en `Presentation/ViewModels/GameViewModel.swift` (huérfano, no en target). Consolidar en el archivo huérfano y agregarlo al target.
- **Duplicación de `AppTheme`**: misma historia entre `MathGameApp.swift` (compilado) y `Presentation/Theme/AppTheme.swift` (huérfano). Consolidar en el archivo huérfano.
- **`DispatchQueue.main.asyncAfter` para feedback** → migrar a `Task { try await Task.sleep(…) }` con cancelación si se desmonta la vista.

### 3.2 Lo que se introduce

- **`@Observable` macro** (iOS 17+) en lugar de `ObservableObject` / `@Published`. Más limpio y eficiente.
- **SwiftData** para persistir `Player` (xp, level, streak, lastPlayedDay, coins), `DailyChallengeRun`, `AchievementProgress`, `BestScore` por modo.
- **Capa Domain pura** (`MathGame/Domain/`): `QuestionGenerator`, `Difficulty`, `Mode`, `Scoring` — sin SwiftUI ni Foundation pesado, testeable en isolation.
- **Targets nuevos en el proyecto**: `MathGame Watch App`, `MathGame Mac` (Catalyst inicialmente; reevaluar nativo si vale la pena).
- **Shared framework** o carpeta compartida con compilación condicional por target para Domain + assets de sonido.
- **Sonido**: AVFoundation con archivos cortos (.caf) precargados, mezcla con respeto al modo silencio del usuario y a la app Música.
- **Haptics**: CoreHaptics para feedback fino (acierto, error, level-up, milestone de racha). Fallback a `UIImpactFeedbackGenerator` en watchOS donde Core Haptics es limitado.
- **Game Center**: GameKit para leaderboards (uno por modo + uno global) y achievements.

### 3.4 Persistencia — modelo `Player`

```swift
@Model final class Player {
    var totalXP: Int
    var level: Int                      // derivado pero cacheado
```

Migración v1→v2: no hay datos persistidos en v1 ⇒ migración trivial (crear `Player` vacío al primer launch de v2).

## 4. Roadmap — 8 semanas

> Cada semana termina con un build **TestFlight verde**. Si una fase se atrasa, el corte se hace en la semana de polish (S8) sacrificando lo más nuevo, nunca lo de fundación.

### Semana 1 · Fundación técnica

- [ ] Resolver duplicaciones de `GameViewModel` y `AppTheme` (en el mismo PR, con tests de smoke).
- [ ] Crear estructura `Domain/Data/Presentation`.
- [ ] Migrar `GameViewModel` a `@Observable`.
- [ ] Migrar callbacks `asyncAfter` a `async/await`.
- [ ] Configurar SwiftData con `Player` mínimo.
- [ ] Limpiar `Localizable.xcstrings` (eliminar entradas `stale`, completar `en` faltantes).
- [ ] Agregar `pt-BR` y `fr` como targets de localización (vacíos por ahora, llenado en S8).
- [ ] CI mínimo: `xcodebuild build` en GitHub Actions.

**Definition of done**: app se ve idéntica al usuario, pero el código compila desde la nueva estructura y persiste `Player` entre lanzamientos.

### Semana 2 · Sistema de progresión

- [ ] Pantalla **Profile** (XP bar, nivel, racha actual, best streak, coins, mejores scores por modo).
- [ ] Cálculo de XP: `+10` por acierto + bonus por racha en sesión.
- [ ] Sistema de niveles (curva XP = `100 * level^1.5`, ajustable).
- [ ] Sistema de **rachas diarias** (jugar el daily ≥1 vez por día). Reset si se pierde un día.
- [ ] **Notificación local** "Tu racha de N días termina en 4h" (con permission flow educado, no al primer launch).
- [ ] Achievements internos (20 al inicio: "Primera victoria", "Racha de 7 días", "100 multiplicaciones perfectas", …).

**DoD**: jugador puede jugar, ganar XP, subir de nivel, mantener racha y ver su progreso.

### Semana 3 · Daily Challenge (motor de retención)

- [ ] Pantalla **Daily** (entrada principal en `ContentView`).
- [ ] Daily Challenge: 10 preguntas mixtas, dificultad creciente, **seed deterministico por fecha** (todos los usuarios ven el mismo daily — esto habilita comparar scores).
- [ ] Resultado del daily: estrellas (1-3), XP bonus, badge "completado hoy".
- [ ] Countdown al próximo daily.
- [ ] Stats del daily (rachas, perfectos, promedio).

**DoD**: hay una razón clara para volver mañana.

### Semana 4 · Polish premium — sonido, haptics, animaciones

- [ ] `AudioEngine` con 8-10 efectos (correct, wrong, level-up, streak-milestone, daily-complete, tap, navigation, achievement-unlocked, time-tick, game-over).
- [ ] Setting para silenciar audio independiente del sistema.
- [ ] `HapticEngine` con patrones (light tap, success, error, milestone, victory).
- [ ] Animaciones spring en transiciones de pregunta, level-up overlay, streak fire animation, XP bar fill.
- [ ] Refinar tipografía (probable: SF Rounded + un display heavy custom para números grandes).
- [ ] Nueva paleta con assets nativos en `.xcassets` (no más fallback hardcoded). Dark mode default + Light mode soportado.
- [ ] Reduce Motion respeta `UIAccessibility.isReduceMotionEnabled`.

**DoD**: la app se siente cara. Una sesión de 5 min es agradable sensorialmente.

### Semana 5 · Nuevos modos + dificultad

- [ ] **Time Attack**: 60 segundos, máximo de respuestas correctas.
- [ ] **Survival**: infinito, dificultad crece, una sola vida.
- [ ] **Mixed**: las 6 operaciones al azar.
- [ ] **Sequence**: ¿qué número sigue? (aritméticas, geométricas, cuadrados).
- [ ] Sistema de **Difficulty** explícito en `Domain/Difficulty/` (Easy/Normal/Hard/Insane) con rangos por modo. Daily usa Normal + ramp.
- [ ] Reorganizar `ContentView` (Daily destacado arriba, modos clásicos en grid, modos nuevos en sección "Más").

**DoD**: hay variedad real, no sólo 6 operaciones iguales con números distintos.

### Semana 6 · iPad

- [ ] Layout adaptativo con `NavigationSplitView` en regular size class.
- [ ] Sidebar permanente con modos a la izquierda, juego a la derecha.
- [ ] Pantalla stats / profile aprovechando ancho.
- [ ] Soporte multitasking (Stage Manager, Slide Over) sin romper.
- [ ] Tipografía y espaciado ajustados (no es iPhone estirado).

**DoD**: en iPad la app se siente nativa, no escalada.

### Semana 7 · Apple Watch + Mac

**watchOS**
- [ ] Target `MathGame Watch App` independiente, comparte `Domain/`.
- [ ] Modo `Quick Math`: 60s, suma/resta, respuestas con `Digital Crown` o picker.
- [ ] Complications: racha actual + best streak (modular small, corner, inline).
- [ ] Haptics nativos del Watch.
- [ ] Sync básico de racha vía `WCSession` (Phone es source of truth).

**Mac (Catalyst)**
- [ ] Habilitar Mac Catalyst en el target principal.
- [ ] Menús nativos (File → New Daily, Game → Restart, View → Profile).
- [ ] Keyboard shortcuts (1-4 para respuestas, ⌘N nuevo daily, ⌘R reset).
- [ ] Window sizing con min/max, no fullscreen forzado.
- [ ] Touch Bar opcional (low priority).

**DoD**: hay app instalable en las 4 plataformas, todas funcionales.

### Semana 8 · Game Center + Lanzamiento

- [ ] **Game Center**: configurar leaderboards (uno por modo + Daily Challenge global) y mapear 20 achievements internos a achievements de Game Center.
- [ ] **Compartir resultado**: generar imagen del resultado del Daily con marca, copiar al portapapeles o share sheet.
- [ ] **Onboarding** (3 pantallas: bienvenida, "tu racha empieza hoy", permisos opcionales).
- [ ] **App Store assets**:
  - Screenshots 6.7" / 6.1" / iPad 13" / iPad 11" / Watch / Mac (Fastlane Snapshot o Rotato).
  - App Preview videos 15-30s para iPhone y iPad.
  - Descripción nueva enfocada en Daily / racha / multi-device.
  - Keywords: "math, brain training, daily challenge, mental math, game".
  - **What's New** en v2.0.0 destacando upgrade gratis para usuarios v1.
- [ ] **Push de versión**: build number, marketing version 2.0.0, archive Release, validate, upload, submit.
- [ ] **TestFlight beta cerrado** durante S6-S7 con 10-20 usuarios; abierto en S8.
- [ ] **Política de privacidad** actualizada (Game Center, notificaciones, no se recoge nada más).

**DoD**: v2.0.0 enviada a revisión y aprobada.

## 6. Riesgos y mitigaciones

| Riesgo | Probabilidad | Impacto | Mitigación |
|---|---|---|---|
| 8 semanas se quedan cortas para 4 plataformas | Alta | Alto | Apple Watch y Mac son las primeras en cortarse si hay atraso; el iPhone+iPad release vale solo. |
| Mac Catalyst se siente "iPad estirado" | Media | Medio | Aceptable en v2.0.0, evaluar AppKit nativo en v2.1. |
| Game Center setup en App Store Connect bloquea release | Media | Medio | Iniciar setup en S1, no en S8. |
| Audio molesta a usuarios silenciosos | Media | Alto | Setting de audio independiente + respetar silent mode. Default `audio off` en primer launch. |
| Notificaciones locales se sienten spam | Alta si mal hechas | Alto | Permission prompt sólo después de 3 días de uso. Una sola notificación al día, después de las 17:00 local. Setting para desactivar. |
| Salto de precio molesta a futuros compradores | Baja | Bajo | Justificable con el cambio mayor; comunicarlo en What's New. |
| Usuarios v1 esperan features que no entran | Media | Bajo | "Upcoming in 2.1" en la pantalla de About. |

## 7. KPIs / criterios de éxito a 30 días post-launch

- Day-1 retention ≥ **50%** (era ~25% típico para apps casuales sin daily).
- Day-7 retention ≥ **25%**.
- Daily Challenge completion rate ≥ **40%** entre usuarios activos.
- Sesiones/día/usuario activo ≥ **2**.
- Rating App Store ≥ **4.6** con ≥ 30 reviews nuevas.
- Crash-free users ≥ **99.5%**.
- Conversion (impresión → compra) en App Store +25% vs v1.
